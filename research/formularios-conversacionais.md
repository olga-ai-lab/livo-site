# Pesquisa: Formulários Conversacionais 2025

## Princípios Typeform (Best Practices)

### 1. Menos é mais
- Formulários com mais de 6 perguntas têm menos de 50% de taxa de conclusão
- Formulários que levam menos de 1 minuto são 15% mais prováveis de serem completados
- Pedir apenas informações críticas

### 2. Fluxo Conversacional
- Apresentar perguntas uma por vez (não bombardear com todas de uma vez)
- Reduz carga cognitiva
- Torna o formulário menos intimidador

### 3. Layout Multi-step
- Dividir em partes digeríveis
- Usuários menos propensos a abandonar se já investiram tempo
- Usar indicadores de progresso

### 4. Lógica Condicional
- Adaptar conteúdo baseado em respostas anteriores
- Mostrar apenas perguntas relevantes
- Personalizar a jornada

### 5. Design Mobile-First
- Caixas de texto maiores
- Botões amigáveis para mobile
- Carregamento rápido

### 6. Smart Defaults e Autocomplete
- Campos pré-preenchidos
- Sugestões de endereço
- Hints de domínio de email
- Usar respostas anteriores em perguntas seguintes (Recall)

### 7. Mensagens de Erro Claras
- Posicionar diretamente abaixo do campo
- Explicar como corrigir

### 8. Microcopy
- Texto curto e útil que guia o usuário
- Explicar por que está pedindo certos dados
- Oferecer reassurance ("Isso nos ajuda a personalizar sua experiência")

## Aplicação para Cotação de Seguros

### Fluxo Proposto (Mínimo de Campos)
1. **Seleção de Produto** - Cards visuais grandes, um clique
2. **Identificação Básica** - Nome + Email (ou CNPJ com auto-preenchimento)
3. **Dados Essenciais** - 2-3 perguntas específicas do produto
4. **Contato** - WhatsApp para retorno

### Design Visual
- Tema CLARO (fundo branco/off-white)
- Uma pergunta por tela
- Transições suaves (fade/slide)
- Cards flutuantes com sombra sutil
- Cores de destaque: Livonius teal (#056677), Livo verde (#7AB72D)
- Tipografia grande e legível
- Botões proeminentes com hover states
- Indicador de progresso minimalista (dots ou barra fina)

### Micro-interações
- Feedback visual ao selecionar opção
- Animação de check ao completar campo
- Transição suave entre etapas
- Loading state elegante
